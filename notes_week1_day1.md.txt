# Why spring boot?
  - it serves as a built-in toolkit for java developers in creating web development and applications
  - no need to setup from scratch.
  - big community so theres a lot of resources online
  - fast development, with less boilerplate code (repetitive codes)
  - good for microservice (small, focused services)
  - easy to deploy


# Features of Spring boot:
  - auto-configuration -> is automatically setting up your applicaitons so that developers can focus on their business logic and codes
  - embedded servers -> no need to install tomcat separately  (web-server) and can run up your app in single comment (java -jar app.jar)
  - REST API creation -> are made easy, no complex configuration needed. (way of two applications to talk over the internet: HTTP METHODS)
  - databae integration -> automatically handles database connections and convert java objects to databases tables (no complex SQL needed for basic operations)
  - security built-in -> keeps your applications protected; password bashing, authentication, and such
  - dependency injection -> provides libraries your codes need
